A* 경로 탐색 알고리즘 분석

2026년 3월 16일
← 블로그 목록으로

1. 왜 A* 알고리즘이 좋은 EE 주제인가

경로 탐색(pathfinding)은 컴퓨터 과학과 수학이 만나는 대표적인 영역입니다. 내비게이션 앱이 최적 경로를 찾는 원리, 게임에서 캐릭터가 장애물을 피해 이동하는 메커니즘, 로봇이 환경을 탐색하는 방법 — 모두 경로 탐색 알고리즘에 기반합니다. 그중 A*(A-star) 알고리즘은 가장 널리 사용되는 최단경로 알고리즘으로, 그래프 이론의 기초 위에 휴리스틱이라는 지능적 탐색 전략을 결합한 것입니다.

A* 알고리즘은 IB EE 주제로서 여러 장점을 가집니다. 첫째, 알고리즘의 수학적 분석(최적성 증명, 시간 복잡도 분석)이 HL 수준에서 접근 가능합니다. 둘째, 직접 구현하여 시각화할 수 있어 개인적 참여를 보여주기 쉽습니다. 셋째, 다른 알고리즘(다익스트라, BFS, 그리디)과의 비교 분석이 자연스러운 연구 구조를 만들어줍니다.

연구 질문 예시: "격자 기반 지도에서 A* 알고리즘의 휴리스틱 함수(맨해튼, 유클리드, 체비셰프)가 탐색 효율성에 미치는 영향을 분석하라."

2. 그래프 이론의 기초

A* 알고리즘을 이해하려면 먼저 그래프 이론의 기본 개념이 필요합니다. 그래프 $G = (V, E)$는 정점(vertex)의 집합 $V$와 간선(edge)의 집합 $E$로 구성됩니다. 경로 탐색 문제에서 정점은 위치(도시, 교차로, 격자 칸)를, 간선은 두 위치를 연결하는 경로를 나타냅니다.

가중 그래프(weighted graph)에서는 각 간선에 비용(cost)이 부여됩니다. 이 비용은 거리, 시간, 또는 다른 양을 나타낼 수 있습니다. 최단경로 문제는 시작 정점 $s$에서 목표 정점 $t$까지의 간선 비용 합이 최소인 경로를 찾는 것입니다.

3. 다익스트라 알고리즘에서 A*로

A*를 이해하기 위해 먼저 다익스트라(Dijkstra) 알고리즘을 살펴봅시다. 다익스트라 알고리즘은 시작점에서 각 정점까지의 최단거리 $g(n)$을 계산합니다. 우선순위 큐(priority queue)를 사용하여 $g(n)$이 가장 작은 정점부터 탐색합니다. 이 알고리즘은 항상 최단경로를 찾지만, 모든 방향으로 균일하게 탐색하므로 비효율적일 수 있습니다.

A*는 다익스트라를 개선한 것입니다. 핵심 아이디어는 "목표 방향으로의 추정 거리"를 추가로 고려하는 것입니다. A*는 다음 평가 함수를 사용합니다:

$$f(n) = g(n) + h(n)$$

여기서 $g(n)$은 시작점에서 정점 $n$까지의 실제 비용이고, $h(n)$은 정점 $n$에서 목표점까지의 추정 비용인 휴리스틱 함수(heuristic function)입니다. A*는 $f(n)$이 가장 작은 정점부터 탐색하므로, 목표 방향으로 편향된 탐색을 수행합니다.

4. 휴리스틱 함수

휴리스틱 함수의 선택은 A* 알고리즘의 성능을 결정합니다. 좋은 휴리스틱은 두 가지 조건을 만족해야 합니다:

  • 허용성(admissibility): $h(n) \leq h^*(n)$ — 실제 최단거리 $h^*(n)$를 과대추정하지 않음. 이 조건이 만족되면 A*는 최적해(최단경로)를 보장합니다.
  • 일관성(consistency): $h(n) \leq c(n, n') + h(n')$ — 삼각부등식을 만족. 일관성은 허용성보다 강한 조건이며, 효율적인 구현을 보장합니다.

격자 기반 지도에서 흔히 사용되는 휴리스틱 함수들을 비교해 봅시다:

$$\text{맨해튼 거리: } h(n) = |x_n - x_t| + |y_n - y_t|$$
$$\text{유클리드 거리: } h(n) = \sqrt{(x_n - x_t)^2 + (y_n - y_t)^2}$$
$$\text{체비셰프 거리: } h(n) = \max(|x_n - x_t|, |y_n - y_t|)$$

4방향 이동만 허용하는 격자에서 맨해튼 거리는 정확한 최단거리의 하한을 제공하며 가장 효율적입니다. 8방향 이동이 가능하면 유클리드 거리나 대각선 거리(octile distance)가 더 적합합니다. 이 선택에 따라 탐색하는 노드 수가 크게 달라지며, 이를 실험적으로 비교하는 것이 EE의 핵심 분석이 됩니다.

5. 시간 복잡도 분석

A*의 시간 복잡도는 휴리스틱의 질에 크게 의존합니다. $h(n) = 0$이면 A*는 다익스트라 알고리즘과 동일하게 작동하며, 시간 복잡도는 $O(|V| \log |V| + |E|)$입니다. 반면 $h(n) = h^*(n)$, 즉 휴리스틱이 실제 최단거리와 정확히 일치하면, A*는 최적 경로 위의 노드만 탐색하여 최대 효율을 달성합니다.

최악의 경우 A*는 그래프의 모든 노드를 탐색해야 할 수도 있지만, 실제 응용에서는 좋은 휴리스틱 덕분에 다익스트라보다 훨씬 적은 노드를 탐색합니다. 이 차이를 실험적으로 측정하고 시각화하는 것이 EE에서 강력한 분석 요소가 됩니다.

6. EE에서 탐구할 수 있는 방향

휴리스틱 함수 비교 실험: 동일한 지도에서 맨해튼, 유클리드, 체비셰프 거리를 사용했을 때의 탐색 노드 수, 경로 길이, 실행 시간을 비교합니다. 장애물의 밀도를 0%, 10%, 20%, 30%, 40%로 변화시키며 실험하면 체계적인 분석이 가능합니다.

다익스트라 vs A* 비교: 격자 크기를 10x10, 20x20, 50x50, 100x100으로 변화시키며 두 알고리즘의 탐색 효율성 차이를 그래프로 나타냅니다. A*가 다익스트라 대비 몇 %의 노드를 절약하는지를 정량적으로 분석하세요.

가중 A*: 평가 함수를 $f(n) = g(n) + w \cdot h(n)$으로 변형하여 가중치 $w$의 영향을 분석합니다. $w > 1$이면 속도는 빠르지만 최적성이 보장되지 않으며, 경로의 품질과 탐색 속도 사이의 trade-off를 실험적으로 보일 수 있습니다.

시각화 제안: A*의 탐색 과정을 격자 위에 색상으로 표현하세요. 탐색 완료된 노드(closed set)를 파란색, 탐색 대기 중인 노드(open set)를 초록색, 최종 경로를 빨간색으로 표시하면 A*의 "지능적 탐색"이 한눈에 드러납니다.

Why ML Digit Recognition Is a Strong EE Topic

Handwritten digit recognition using the MNIST dataset is often called the "hello world" of machine learning. It is the problem that every machine learning practitioner solves first, and for good reason: the dataset is clean, the task is well-defined, the results are measurable, and the mathematical concepts underlying the solution are rich enough to sustain a 4,000-word investigation.

For an IB Extended Essay, this topic offers a rare combination of accessibility and depth. The basic task — training a model to recognize digits from 0 to 9 in 28x28 pixel grayscale images — can be accomplished with a few lines of Python using TensorFlow or scikit-learn. But the mathematical questions behind why the model works, how different architectures affect performance, and what the model gets wrong are substantial enough to demonstrate genuine analytical thinking.

This topic works as either a Mathematics EE or a Computer Science EE. If you submit it as Mathematics, the emphasis should be on the linear algebra (matrix operations, weight optimization), calculus (gradient descent, backpropagation), and statistical analysis (accuracy metrics, confusion matrices). If you submit it as Computer Science, the emphasis shifts toward architecture decisions, computational efficiency, and algorithmic comparison. I recommend Mathematics for students taking AA HL, and Computer Science for students who want to showcase programming skill.

Why MNIST specifically? The MNIST dataset contains 70,000 labeled images (60,000 training, 10,000 test). It is universally available, requires no data cleaning, and has decades of published benchmark results for comparison. This means you can focus your entire essay on mathematical analysis rather than data wrangling.

Sample Research Questions

  • "How does the architecture of a neural network affect its accuracy in classifying handwritten digits from the MNIST dataset?" — The core question. You implement multiple architectures (single-layer perceptron, two-layer network, three-layer network) and compare their performance systematically. Clear, focused, and naturally generates quantitative results.
  • "To what extent does the number of hidden neurons in a single hidden layer affect the classification accuracy and training time for MNIST digit recognition?" — More narrow. You fix the architecture (one hidden layer) and vary the number of neurons (16, 32, 64, 128, 256, 512). Plots accuracy and training time as functions of network width.
  • "How does the choice of activation function (sigmoid, tanh, ReLU) affect the convergence speed and final accuracy of a neural network trained on the MNIST dataset?" — Mathematically rich because each activation function has different derivative properties that affect gradient flow.
  • "What patterns emerge in the misclassification of handwritten digits by a neural network, and can confusion matrix analysis explain why certain digit pairs are more frequently confused?" — Analytical focus. You train one good model and then spend most of the essay analyzing its errors. Particularly strong for critical thinking (Criterion C).

Key Mathematical Concepts

Neural Networks as Mathematical Functions

A neural network is a parameterized mathematical function that maps inputs to outputs through a series of linear transformations and nonlinear activations. For a single hidden layer with input vector x, weight matrices W1 and W2, bias vectors b1 and b2, and activation function f:

output = softmax(W2 · f(W1 · x + b1) + b2)

The network learns by adjusting W1, W2, b1, and b2 to minimize the difference between predicted and actual outputs across all training examples.

For MNIST, the input x is a 784-dimensional vector (28 x 28 pixels flattened), and the output is a 10-dimensional vector representing the probability of each digit 0–9. The softmax function converts raw output scores into probabilities that sum to 1.

Supervised Learning and the Training Process

Supervised learning means the model learns from labeled examples. Each training image has a known correct label (the digit it represents). The training process iteratively adjusts the network weights to minimize a loss function — typically cross-entropy loss for classification:

L = -Σ yi log(ŷi)

where yi is the true label (1 for the correct class, 0 for others) and ŷi is the predicted probability. This loss function has a clean mathematical interpretation: it measures the information-theoretic distance between the true distribution and the predicted distribution.

Gradient Descent and Backpropagation

The network learns through gradient descent: compute the gradient of the loss function with respect to every weight, then update each weight in the direction that reduces the loss. Backpropagation is the algorithm that efficiently computes these gradients using the chain rule of calculus, propagating error signals backward from the output layer through the hidden layers to the input.

For a Mathematics EE, explaining backpropagation for a small network (e.g., 2 inputs, 2 hidden neurons, 1 output) with explicit numerical examples is an excellent way to demonstrate mathematical understanding. Show the forward pass, compute the loss, then trace the chain rule backward through each layer to compute the gradient of each weight.

Training/Test Split and Overfitting

The training data is used to learn the weights; the test data is used to evaluate performance on unseen examples. Overfitting occurs when the model memorizes the training data instead of learning general patterns — it performs well on training data but poorly on test data. The gap between training accuracy and test accuracy is a direct measure of overfitting, and analyzing this gap for different architectures provides quantitative evidence for your analysis.

Accuracy Metrics and Confusion Matrices

Overall accuracy (percentage of correctly classified test images) is the simplest metric, but it hides important details. A confusion matrix is a 10x10 grid showing, for each true digit, how many times the model predicted each digit. This reveals which digit pairs the model confuses most frequently — typically 4/9, 3/5, and 7/1 — and provides material for mathematical analysis of why those confusions occur (visual similarity of certain handwriting styles).

Python + TensorFlow/KerasNeural networks with full control over architectureMedium — more verbose but very flexible

For most IB students, Keras (the high-level API for TensorFlow) is the best choice. It provides clean, readable code that you can include in your essay and appendix without overwhelming the reader:

# Basic neural network for MNIST (Keras)
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Flatten(input_shape=(28, 28)),
    layers.Dense(128, activation='relu'),
    layers.Dense(10, activation='softmax')
])

model.compile(optimizer='adam',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])

model.fit(x_train, y_train, epochs=10, validation_split=0.1)
test_loss, test_acc = model.evaluate(x_test, y_test)

Keeping the Focus on Mathematics

The biggest mistake students make with ML-based Extended Essays is writing a programming tutorial instead of a mathematical investigation. Your essay is not about how to use Keras. It is about what the mathematics reveals.

Do this: Explain the mathematical operation at each layer. Show that the first layer performs a matrix multiplication of a 784-dimensional vector by a 784x128 weight matrix, producing a 128-dimensional hidden representation. Explain what the ReLU activation does mathematically (f(x) = max(0, x)) and why this nonlinearity is essential (without it, the entire network reduces to a single linear transformation, regardless of depth).

Do this: Derive the gradient of the loss function for a small example. Even a 2-input, 2-hidden, 2-output network with explicit numbers shows that you understand backpropagation, not just that you can call model.fit().

Do this: Analyze the confusion matrix mathematically. If your model confuses 4 and 9 fifteen times more often than 0 and 1, quantify this ratio and explain it in terms of pixel overlap between the average 4 and average 9 in the dataset.

Do not do this: Spend 1,000 words explaining how to install Python and TensorFlow. Do not include full code listings in the body of the essay (put them in an appendix). Do not describe the training process as a black box ("the model learned to recognize digits").

The IB examiner's perspective: An examiner reading a Mathematics EE wants to see mathematical reasoning on every page. If three consecutive paragraphs contain no equations, no mathematical notation, and no quantitative analysis, you are drifting into description and losing marks on Criterion C.

How to Structure This EE

  1. Introduction (400 words): Context of digit recognition and machine learning, research question, overview of approach.
  2. Mathematical Background (800 words): Neural network as a mathematical function, activation functions and their derivatives, loss function definition, gradient descent, backpropagation with a small worked example.
  3. Methodology (500 words): Dataset description (MNIST), architectures to be compared, hyperparameter choices, training/test split rationale, hardware and software used.
  4. Results (900 words): Training curves (loss and accuracy vs. epoch) for each architecture, test accuracy comparison table, confusion matrices, analysis of misclassified examples.
  5. Analysis and Discussion (800 words): Why deeper/wider networks perform better (or do not), overfitting analysis, which digits are hardest to classify and why, comparison with published benchmarks.
  6. Conclusion (400 words): Direct answer to research question, what the investigation reveals about the relationship between neural network architecture and classification performance.

Expected Grade Potential

ML digit recognition is a reliable A-grade topic when the mathematical emphasis is maintained throughout. The topic provides clear experimental results, natural opportunities for quantitative analysis, and rich material for critical discussion. The IB examiner will have seen many ML essays, so what distinguishes an A from a B is the depth of mathematical reasoning — not the sophistication of the code.

Students who score As on this topic typically include a backpropagation derivation by hand, present confusion matrix analysis with quantitative explanations, and discuss the mathematical reasons for their results (not just "network A was better than network B" but "the additional hidden layer allows the network to learn hierarchical features — edge detectors in layer 1 that combine into digit-part detectors in layer 2 — which reduces the test error from 3.2% to 1.8%").
